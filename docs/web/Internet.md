## 互联网

互联网是由使用 TCP/IP 协议互联的计算机组成的全球网络。

请求首先通过互联网服务提供商 (ISP) 发送到域名系统 (DNS) 服务器进行域名到 IP 地址的转换，然后通过路由器在网络中路由到目标服务器。它实现了动态的、分散的全球通信。

所有这些网络连接在一起就构成了互联网。

#### 互联网如何运作：概述

从宏观层面来看，互联网的工作原理是利用一套标准化的协议将设备和计算机系统连接起来。这些协议定义了设备之间信息交换的方式，并确保数据传输的可靠性和安全性。

互联网上的信息以比特的形式，通过各种介质（包括以太网电缆、光纤电缆和无线信号（即无线电波））从一台计算机传输到另一台计算机。

互联网的核心是一个由相互连接的路由器组成的全球网络，这些路由器负责在不同设备和系统之间引导流量。当您通过互联网发送数据时，数据会被分解成许多小数据包，这些数据包从您的设备发送到路由器。路由器检查数据包并将其转发到通往目的地的下一个路由器。这个过程会一直持续到数据包到达最终目的地为止。

#### 基本概念和术语

要了解互联网，熟悉一些基本概念和术语非常重要。以下是一些需要了解的关键术语和概念：

- **数据包：**通过互联网传输的小型数据单元。
- **路由器：**一种在不同网络之间传输数据包的设备。
- **IP地址：**分配给网络上每个设备的唯一标识符，用于将数据路由到正确的目的地。
- **域名：**用于识别网站的人类可读名称，例如 google.com。
- **DNS：**域名系统负责将域名转换为 IP 地址。
- **HTTP：**超文本传输协议用于在客户端（例如网络浏览器）和服务器（例如网站）之间传输数据。
- **HTTPS：** HTTP 的加密版本，用于在客户端和服务器之间提供安全通信。
- **SSL/TLS：**安全套接字层协议和传输层安全协议用于在互联网上提供安全通信。

#### 协议在互联网中的作用

协议在实现互联网通信和数据交换方面发挥着至关重要的作用。协议是一套规则和标准，用于定义设备和系统之间信息交换的方式。

互联网通信中使用了许多不同的协议，包括互联网协议（IP）、传输控制协议（TCP）、用户数据报协议（UDP）、域名系统（DNS）等等。

IP协议负责将数据包路由到正确的目的地，而TCP和UDP协议则确保数据包可靠高效地传输。DNS用于将域名解析为IP地址，HTTP用于在客户端和服务器之间传输数据。

使用标准化协议的关键优势之一在于，它允许不同制造商和供应商的设备和系统之间无缝通信。

#### 了解 IP 地址和域名

在进行互联网活动时，理解 IP 地址和域名都是非常重要的概念。

IP 地址是分配给网络上每个设备的唯一标识符。它用于将数据路由到正确的目的地，确保信息发送给预期的接收者。IP 地址通常表示为四个用句点分隔的数字，例如“192.168.1.1”。

域名则是用于识别网站和其他互联网资源的易于理解的名称。它们通常由两个或多个部分组成，部分之间用句点分隔。例如，“google.com”就是一个域名。域名通过域名系统（DNS）转换为IP地址。

DNS是互联网基础设施的关键组成部分，负责将域名转换为IP地址。当您在浏览器中输入域名时，您的计算机会向DNS服务器发送DNS查询，服务器会返回相应的IP地址。然后，您的计算机使用该IP地址连接到您请求的网站或其他资源。

#### HTTP 和 HTTPS 简介

HTTP（超文本传输协议）和 HTTPS（安全 HTTP 协议）是基于互联网的应用程序和服务中最常用的两种协议。

HTTP 是用于在客户端（例如 Web 浏览器）和服务器（例如网站）之间传输数据的协议。当您访问网站时，您的 Web 浏览器会向服务器发送 HTTP 请求，请求您所需的网页或其他资源。然后，服务器会将包含所请求数据的 HTTP 响应发送回客户端。

HTTPS 是 HTTP 的更安全版本，它使用 SSL/TLS（安全套接字层/传输层安全）加密技术对客户端和服务器之间传输的数据进行加密。这提供了一层额外的安全保障，有助于保护登录凭证、支付信息和其他个人数据等敏感信息。

当您访问使用 HTTPS 协议的网站时，您的浏览器地址栏会显示一个挂锁图标，表明连接是安全的。您可能还会看到网站地址开头是“https”，而不是“http”。

#### 使用 TCP/IP 构建应用程序

TCP/IP（传输控制协议/互联网协议）是大多数基于互联网的应用程序和服务所使用的底层通信协议。它能够确保运行在不同设备上的应用程序之间可靠、有序且经过错误检查地传输数据。

使用 TCP/IP 构建应用程序时，需要理解以下几个关键概念：

- 端口：端口用于识别设备上运行的应用程序或服务。每个应用程序或服务都被分配一个唯一的端口号，从而确保数据能够发送到正确的目的地。
- 套接字：套接字是 IP 地址和端口号的组合，代表特定的通信端点。套接字用于在设备之间建立连接，并在应用程序之间传输数据。
- 连接：当两个设备想要相互通信时，会在两个套接字之间建立连接。在连接建立过程中，设备会协商各种参数，例如最大段大小和窗口大小，这些参数决定了数据如何通过连接传输。
- 数据传输：连接建立后，即可在各设备上运行的应用程序之间传输数据。数据通常以数据段的形式传输，每个数据段包含序列号和其他元数据，以确保可靠传输。

使用 TCP/IP 构建应用程序时，您需要确保应用程序的设计能够与正确的端口、套接字和连接兼容。您还需要熟悉 TCP/IP 常用的各种协议和标准，例如 HTTP、FTP（文件传输协议）和 SMTP（简单邮件传输协议）。

#### 使用 SSL/TLS 保护互联网通信

正如我们之前讨论的，SSL/TLS 是一种用于加密通过互联网传输的数据的协议。它通常用于为网页浏览器、电子邮件客户端和文件传输程序等应用程序提供安全连接。

使用 SSL/TLS 保护互联网通信时，需要理解以下几个关键概念：

- **证书：** SSL/TLS 证书用于在客户端和服务器之间建立信任关系。它们包含有关服务器身份的信息，并由受信任的第三方（证书颁发机构）签名以验证其真实性。
- **握手：**在 SSL/TLS 握手过程中，客户端和服务器交换信息，以协商加密算法和其他参数，从而建立安全连接。
- **加密：**一旦建立安全连接，数据就会使用约定的算法进行加密，并且可以在客户端和服务器之间安全传输。

在构建基于互联网的应用程序和服务时，了解 SSL/TLS 的工作原理至关重要，并确保您的应用程序在传输敏感数据（例如登录凭证、支付信息和其他个人数据）时使用 SSL/TLS。您还需要确保为服务器获取并维护有效的 SSL/TLS 证书，并遵循配置和保护 SSL/TLS 连接的最佳实践。这样做有助于保护用户数据，并确保应用程序通过互联网通信的完整性和机密性。

什么是HTTP？

超文本传输协议 (HTTP) 是万维网的基础，用于通过超文本链接加载网页。HTTP 是一种[应用层](https://www.cloudflare.com/learning/ddos/application-layer-ddos-attack/)协议，通过[TCP](https://developer.mozilla.org/en-US/docs/Glossary/TCP)或[TLS](https://developer.mozilla.org/en-US/docs/Glossary/TLS)加密的TCP连接传输数据，并运行在网络[协议](https://www.cloudflare.com/learning/network-layer/what-is-a-protocol/)栈的其他层之上。典型的 HTTP 通信流程是：客户端向服务器发出请求，服务器随后发送响应消息。



客户端：用户代理
用户代理是代表用户行为的任何工具。这一角色主要由网络浏览器承担，但也可能是工程师和Web开发人员用来调试应用程序的程序。

浏览器始终是发起请求的实体，服务器永远不会主动发起请求（尽管近年来已添加了一些机制来模拟服务器主动发起的消息）。

要显示一个网页，浏览器首先发送初始请求以获取代表该页面的HTML文档。接着，浏览器解析该文件，并发出额外的请求来获取对应的执行脚本、显示所需的布局信息（CSS）以及页面中包含的子资源（通常是图像和视频）。然后，网络浏览器将这些资源组合起来，呈现完整的文档——即网页。浏览器执行的脚本可能会在后续阶段获取更多资源，浏览器也会相应地更新网页。

网页是一种超文本文档。这意味着所显示的部分内容包含链接，用户可以通过激活这些链接（通常通过鼠标点击）来获取新的网页，从而引导其用户代理在网络中进行导航。浏览器将这些操作指令转换为HTTP请求，并进一步解析HTTP响应，以向用户呈现清晰的响应结果。



Web 服务器
在通信通道的另一端是服务器，它负责响应客户端的请求并提供文档。从表面看，服务器似乎只是一台独立的机器；但实际上，它可能是一组共同分担负载的服务器集群（负载均衡），也可能包含其他软件（如缓存系统、数据库服务器或电子商务服务器），能够按需完全或部分动态生成文档。

服务器不一定对应单台物理机器，多个服务器软件实例可以部署在同一台机器上。借助 HTTP/1.1 及其 [Host](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Host) 头部功能，这些不同实例甚至可以共享同一个 IP 地址。



代理
在 Web 浏览器和服务器之间，有许多计算机和设备负责转发 HTTP 消息。由于 Web 协议栈的分层结构，这些中间设备大多工作在传输层、网络层或物理层，对 HTTP 层是透明的，但它们可能对性能产生重要影响。那些工作在应用层的中间设备通常被称为**代理**。代理可以是透明的（即在不修改请求的情况下直接转发接收到的请求），也可以是非透明的（即在将请求传递给服务器之前以某种方式修改请求）。代理可以实现多种功能：

- **缓存**（缓存可以是公共的，也可以是私有的，如浏览器缓存）
- **过滤**（如病毒扫描或家长控制）
- **负载均衡**（使多台服务器能够处理不同的请求）
- **认证**（控制对不同资源的访问）
- **日志记录**（允许存储历史信息）



统一资源定位符（URL）

URL（统一资源定位符）用于在网络上唯一标识一个资源。URL 的语法如下：

```
协议://主机名:端口/路径和文件名
```

URL 由 4 个部分组成：

1. *协议*：客户端和服务器使用的应用层协议，例如 HTTP、FTP 和 telnet。
2. *主机名*`www.nowhere123.com`：服务器的DNS 域名（例如， ）或 IP 地址（例如，192.128.1.2）。
3. *端口*：服务器监听来自客户端的传入请求的 TCP 端口号。URL 中未指定端口号，使用默认值，即 HTTP 的 TCP 端口 80
4. *路径和文件名*：请求资源的名称和位置，位于服务器文档基本目录下。



HTTP 是无状态的，但并非无会话的。

HTTP 协议是无状态的：在同一连接上连续执行的两个请求之间没有任何关联。这很容易导致用户在与某些页面进行连贯交互时遇到问题，例如使用电子商务购物车。虽然 HTTP 协议的核心是无状态的，但 HTTP Cookie 允许使用有状态的会话。通过扩展 HTTP 头部，可以将 HTTP Cookie 添加到工作流程中，从而允许在每个 HTTP 请求上创建会话，使会话共享相同的上下文或状态。



HTTP 可以控制什么

HTTP 的这种可扩展性随着时间的推移，使得 Web 拥有了更多控制权和功能。缓存和身份验证机制在 HTTP 发展初期就已经实现。相比之下，放宽*源限制的能力直到 2010 年代才被添加进来。*

以下是可通过 HTTP 控制的常见功能列表：

- *[缓存](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Caching)*：HTTP 可以控制文档的缓存方式。服务器可以指示代理和客户端缓存哪些文档以及缓存多长时间。客户端可以指示中间缓存代理忽略已存储的文档。
- *放宽来源限制*：为了防止窥探和其他侵犯隐私的行为，Web 浏览器强制执行网站之间的严格隔离。只有来自**同一来源的**页面才能访问网页的全部信息。虽然这种限制会给服务器带来负担，但 HTTP 标头可以在服务器端放宽这种严格的隔离，允许文档成为来自不同域的信息的拼接体；这样做甚至可能出于安全方面的考虑。
- *身份验证*：某些页面可能受到保护，只有特定用户才能访问。基本身份验证可以通过 HTTP 提供，可以使用[`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Cookies)[`WWW-Authenticate`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/WWW-Authenticate) http://example.com/ ...
- *[代理和隧道](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Proxy_servers_and_tunneling)*：服务器或客户端通常位于内网中，并对其他计算机隐藏其真实 IP 地址。HTTP 请求随后通过代理服务器跨越这道网络屏障。并非所有代理都是 HTTP 代理。例如，SOCKS 协议运行在更底层。其他协议，例如 FTP，也可以由这些代理处理。
- *会话*：使用 HTTP Cookie 可以将请求与服务器状态关联起来。尽管基本的 HTTP 协议是无状态的，但这种方法也能创建会话。这不仅适用于电子商务购物车，也适用于任何允许用户配置输出的网站。

HTTP 请求中包含什么内容？

HTTP 请求是网络浏览器等互联网通信平台请求加载网站所需信息的方式。

通过互联网发出的每个 HTTP 请求都携带一系列编码数据，这些数据包含不同类型的信息。一个典型的 HTTP 请求包含：

```
请求方法名 请求URI  HTTP版本
请求头名称：请求头值1、请求头值2、……
```

1. HTTP 版本类型
2. 网址
3. HTTP 方法
4. HTTP 请求头
5. 可选的HTTP请求体。

当此请求消息到达服务器时，服务器可以采取以下任一操作：

服务器解析收到的请求，将请求映射到服务器文档目录下的一个文件中，并将请求的文件返回给客户端。
服务器解释收到的请求，将请求映射到服务器端保存的程序，执行该程序，并将程序的输出返回给客户端。
请求无法满足，服务器返回错误信息。



什么是HTTP方法？

HTTP 方法（有时也称为 HTTP 动词）指示 HTTP 请求期望从被查询的服务器执行的操作。例如，两种最常见的 HTTP 方法是“GET”和“POST”；“GET”请求期望服务器返回信息（通常是网站页面），而“POST”请求通常表示客户端正在向 Web 服务器提交信息（例如表单信息，如提交的用户名和密码）。



什么是HTTP请求头？
HTTP 标头包含以键值对形式存储的文本信息，每个 HTTP 请求（以及响应，稍后会详细介绍）都会包含这些标头。这些标头传递核心信息，例如客户端使用的浏览器以及请求的数据。



HTTP响应中包含哪些内容？
HTTP响应是Web客户端（通常是浏览器）从互联网服务器收到的、针对HTTP请求的回复。这些响应根据HTTP请求中请求的内容传递有价值的信息。

典型的HTTP响应包含：

```
HTTP 版本 状态码 原因短语
响应头名称：响应头值1、响应头值2、……
```

HTTP响应体中包含哪些内容？

成功的 HTTP GET 请求响应通常包含一个请求体，其中包含所请求的信息。在大多数 Web 请求中，这是 HTML 数据，Web 浏览器会将其转换为网页。



什么是HTTP状态码？
HTTP 状态码是三位数的代码，通常用于指示 HTTP 请求是否已成功完成。状态码分为以下 5 个部分：

- 1xx（信息）：请求已收到，服务器正在继续处理。
- 2xx（成功）：请求已成功接收、理解、接受和处理。
- 3xx（重定向）：必须采取进一步措施才能完成请求。
- 4xx（客户端错误）：客户端错误, 请求包含错误的语法或无法理解。
- 5xx（服务器错误）：服务器端出现错误, 服务器未能满足看似有效的请求。

“xx”指的是00到99之间的不同数字。

一些常见的状态码包括：

- 100 继续：服务器已收到请求，正在给出响应。
- 200 OK：请求已完成。
- 301 永久迁移：请求的资源已永久迁移到新位置。新位置的 URL 包含在名为 `<response-header-name>` 的响应头中`Location`。客户端应向新位置发出新的请求。应用程序应更新所有指向此新位置的引用。
- 302 Found & Redirect（或临时重定向）：与 301 状态码相同，但新位置是临时的。客户端需要发出新的请求，但应用程序无需更新引用。
- 304 未修改：服务器响应`If-Modified-Since`条件 GET 请求，通知所请求的资源尚未被修改。
- 400 错误请求：服务器无法解释或理解请求，可能是请求消息中存在语法错误。
- 401 身份验证要求：请求的资源受到保护，需要客户端的凭据（用户名/密码）。客户端应使用其凭据（用户名/密码）重新提交请求。
- 403 禁止访问：服务器拒绝提供资源，无论客户端身份如何。
- 404 未找到：服务器上找不到请求的资源。
- 405 方法不允许：请求中使用的方法（例如 POST、PUT、DELETE）是有效的方法。但是，服务器不允许使用该方法访问所请求的资源。
- 408 请求超时：
- 414 请求 URI 过大：
- 500 内部服务器错误：服务器出现混乱，通常是由于服务器端程序响应请求时出现错误所致。
- 501 方法未实现：使用的请求方法无效（可能是由于输入错误造成的，例如，“GET”拼写错误为“Get”）。
- 502 Bad Gateway：代理或网关表示从上游服务器收到错误响应。
- 503 服务不可用：服务器由于过载或维护而无法响应。客户端可以稍后重试。
- 504 网关超时：代理或网关表示从上游服务器收到超时信息。



DDoS攻击可以通过HTTP发起吗？
请记住，HTTP 是一种“无状态”协议，这意味着每个命令都是独立于其他命令运行的。在最初的规范中，每个 HTTP 请求都会创建和关闭一个TCP连接。在较新版本的 HTTP 协议（HTTP 1.1 及更高版本）中，持久连接允许多个 HTTP 请求通过一个持久的 TCP 连接进行传输，从而提高资源利用率。在DoS或DDoS 攻击的背景下，大量 HTTP 请求可用于对目标设备发起攻击，并且被视为应用层攻击或第 7 层攻击的一部分

## 域名

域名是互联网基础设施的关键组成部分。它们为互联网上的任何网络服务器提供了一个易于理解的地址。

[任何连接到互联网的计算机都可以通过公共IP 地址](https://developer.mozilla.org/en-US/docs/Glossary/IP_Address)访问，可以是 IPv4 地址（例如，`192.0.2.172`）或 IPv6 地址（例如，`2001:db8:8b73:0000:0000:8a2e:0370:1337`）。

网站的实际地址是一个复杂的数字 IP 地址（例如 192.0.2.2），但得益于[DNS 服务](https://www.cloudflare.com/learning/dns/what-is-dns/)，用户只需输入易于理解的域名，即可访问他们想要的网站。这个过程称为 DNS 查询。

域名结构简单，由几个部分组成（可能只有一个部分，也可能有两个部分，三个部分……），各部分之间用点号分隔，并且从右到左读取：

- [TLD](https://developer.mozilla.org/en-US/docs/Glossary/TLD)（顶级域名）。

  顶级域名 (TLD) 向用户传达域名背后服务的总体用途。最通用的顶级域名（例如`.com`.NET、`.org`.NET、.NET `.net`）对 Web 服务没有特定要求，但有些顶级域名会执行更严格的策略，以便更清晰地说明其用途。例如：本地顶级域名（例如`.us`、`.fr`或 ）`.se`可能要求服务以特定语言提供或托管在特定国家/地区——它们应该指示特定语言或国家/地区的资源。包含特定域名的顶级域名`.gov`仅允许政府部门使用。该`.edu`顶级域名仅供教育和学术机构使用。顶级域名可以包含特殊字符以及拉丁字符。顶级域名的最大长度为 63 个字符，但大多数顶级域名只有 2 到 3 个字符左右。完整的顶级域名列表[由 ICANN 维护](https://www.icann.org/en/contracted-parties/registry-operators/resources/list-of-top-level-domains)。

- [标签（或组件）](https://developer.mozilla.org/en-US/docs/Learn_web_development/Howto/Web_mechanics/What_is_a_domain_name#label_or_component)

  标签是顶级域名 (TLD) 之后的内容。标签是一个不区分大小写的字符序列，长度为 1 到 63 个字符，仅包含字母`A`1到 0 `Z`、数字`0`1 到 1`9`以及连字符“-”（连字符不能是标签的第一个或最后一个字符）。例如，`a``a`、`97``b` 和 `c``hello-strange-person-16-how-are-you`都是有效的标签示例。位于顶级域名 (TLD) 之前的标签也称为*二级域名*(SLD)。域名可以包含多个标签（或组成部分）。域名并非必须包含三个标签才能构成。例如，[informatics.ed.ac.uk](https://informatics.ed.ac.uk/)就是一个有效的域名。对于您控制的任何域名（例如[mozilla.org](https://www.mozilla.org/en-US/)），您可以创建多个子域名，每个子域名可以包含不同的内容，例如[developer.mozilla.org](https://developer.mozilla.org/)、[support.mozilla.org](https://support.mozilla.org/)或[bugzilla.mozilla.org](https://bugzilla.mozilla.org/)。



域名和网址有什么区别？

统一资源定位符（URL），有时也称为网址，包含网站的域名以及其他信息，包括协议和路径。例如，在 URL“https://cloudflare.com/learning/”中，“cloudflare.com”是域名，“ [https](https://www.cloudflare.com/learning/ssl/what-is-https/) ”是协议，“/learning/”是指向网站上特定页面的路径。

#### 获取域名

这个过程非常简单：

1. 访问注册机构的网站。
2. 通常会有一个醒目的“获取域名”行动号召按钮。点击它。
3. 请填写表格并提供所有必要信息。尤其要确保您没有拼错您想要的域名。一旦付款，就为时已晚了！
4. 域名注册成功后，注册商会通知您。几个小时内，所有 DNS 服务器都会收到您的 DNS 信息。



#### DNS刷新

DNS 数据库存储在全球每个 DNS 服务器上，所有这些服务器都指向一些特殊的服务器，这些服务器被称为“权威名称服务器”或“顶级 DNS 服务器”——它们就像管理系统的老板服务器。

每当您的域名注册商创建或更新某个域名的信息时，所有 DNS 数据库中的信息都必须更新。每个了解该域名的 DNS 服务器都会将信息存储一段时间，之后信息会自动失效并更新（DNS 服务器会查询权威服务器并从中获取更新后的信息）。因此，了解该域名的 DNS 服务器需要一些时间才能获取到最新信息







## 主机

主机托管服务提供服务器空间和资源，用于存储网站并通过互联网进行交付。主机托管类型包括共享主机、虚拟专用服务器 (VPS)、独立主机和具有可扩展资源的云主机。服务内容包括基础设施、域名注册、安全保障和技术支持，以确保网站可靠可用。



### [网页](https://developer.mozilla.org/en-US/docs/Learn_web_development/Getting_started/Environment_setup/Browsing_the_web#web_page)

网页是浏览器可以显示的简单文档。网页可以嵌入各种不同类型的资源，例如**：**

- *样式信息*——控制页面的外观和感觉。
- *脚本*——为页面添加交互功能。
- *媒体*——图像、声音和视频。

**注意：** 浏览器还可以显示其他文档，例如[PDF](https://developer.mozilla.org/en-US/docs/Glossary/PDF)文件和其他资源，例如图像或视频，但**“网页”**一词特指 HTML 文档。

所有网页都有一个唯一的地址（网址，也称为[URL](https://developer.mozilla.org/en-US/docs/Glossary/URL)）。要访问某个网页，只需在浏览器地址栏中输入其地址即可：

### [网站](https://developer.mozilla.org/en-US/docs/Learn_web_development/Getting_started/Environment_setup/Browsing_the_web#website)

网站是由一系列链接的网页（及其相关资源）组成的集合，这些网页共享一个唯一的*域名*[。](https://developer.mozilla.org/en-US/docs/Learn_web_development/Howto/Web_mechanics/What_is_a_domain_name)给定网站的每个网页都提供明确的链接——通常以可点击的文本形式出现——使用户能够从网站的一个页面跳转到另一个页面。



### [Web服务器](https://developer.mozilla.org/en-US/docs/Learn_web_development/Getting_started/Environment_setup/Browsing_the_web#web_server)

网络*服务器*是一台托管一个或多个*网站的*计算机。“托管”意味着所有*网页*及其相关文件都位于该计算机上。当用户尝试加载网页时，*网络服务器*会将托管的网页文件发送到用户的浏览器。

不要将*网站*和*网络服务器*混淆。例如，如果你听到有人说“我的网站没有响应”，这很可能意味着*网络服务器*没有响应，因此*网站*无法访问。

*更重要的是，由于一台网络服务器可以托管多个网站，因此“网络服务器”*一词不再用于指代单个网站，以免造成混淆。如果有人说“我的网络服务器没有响应”，则可能是托管在该网络服务器上的多个网站或应用程序都无法访问。
