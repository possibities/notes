### 前言

#### 如何成为一名后端开发人员

如果你是完全零基础的初学者，可以从学习一门后端编程语言开始，例如 Python、Ruby、Java、Go 等。在掌握了该语言的基础到中级知识后，学习该语言的包管理工具，了解如何安装并在你的应用中使用外部包。学习某种关系型数据库（如 PostgreSQL）的基础知识，掌握如何执行简单的 CRUD 操作。此外，你也可以选择并学习一门与你所选语言配套的 Web 框架。学习如何构建一个简单的 RESTful API，并为其实现基本的身份验证/授权功能。在学习上述所有内容时，别忘了同时学习 Git 和 GitHub。

完成以上所有学习步骤后，你就可以开始申请初级后端开发岗位了。同时，参考上文的后端开发路线图，以了解整体技术全景，并查漏补缺。计算机科学或相关领域的学位并非必需，但积极进行社交、构建个人作品集、主动寻找实习机会、初级开发岗位或咨询工作，都有助于你开启并推进后端开发职业之路。

注：学习过程中务必通过大量项目实践来巩固对概念的理解。同时，保持持续学习的态度至关重要，这能帮助你提升技能，并适应行业快速发展的技术演进。

#### 成为一名后端开发人员需要多久

成为一名后端开发者所需的时间因人而异，这取决于多种因素，例如你的学习节奏、已有经验以及能够投入学习的时间。

如果你有计算机科学或相关领域的背景，并且具备编程经验，可能会在相对较短的时间内——可能只需要几个月——转型为后端开发者。然而，如果你在计算机科学领域经验或教育背景较少甚至为零，则可能需要更长时间来掌握必要的技能、积累成为熟练后端开发者所需的经验。这个过程可能需要6个月到2年不等。



#### 后端开发人员的薪资水平

后端开发者的薪资水平取决于多种因素，例如工作地点、经验背景以及公司规模。根据 Glassdoor 的数据，美国后端开发者的平均基本年薪约为 92,000 美元。然而，这一数字在不同地区差异显著——在旧金山、西雅图和纽约等高薪资城市，平均年薪可达 120,000 至 135,000 美元。

需要留意的是，以上仅为平均数据。实际薪资会因经验水平、具体技能和就职公司等因素产生较大波动。随着经验积累及掌握与目标职位更匹配的专业技能，你的收入有望进一步提升。

建议通过多种渠道获取信息，包括薪资调查报告、招聘网站等，以了解当前市场在你所在地区和经验水平下的普遍情况。同时，也可以尝试联系该领域的其他专业人士，了解他们的实际经验和薪资范围。

#### 后端框架是什么

后端框架是驱动 Web 应用服务器端的核心框架，负责管理数据、处理请求并生成返回至客户端的响应。

目前最流行的后端框架包括 Python 的 Django、Ruby 的 Ruby on Rails、Java 的 Spring Boot 以及 Go 的 Gin。

这些框架通常与 PostgreSQL、MySQL、MongoDB 等数据库，以及 Docker、Kubernetes、Jenkins 等工具协同工作。后端框架与其他服务器端组件共同构建起 Web 应用的功能体系，支撑用户最终在客户端交互的体验。

### 后端开发工程职位描述

后端开发是指Web开发的服务器端部分，主要负责创建和管理服务器逻辑、数据库和API。它涉及处理用户身份验证、授权和用户请求，通常使用Python、Java、Ruby、PHP、JavaScript（Node.js）和.NET等[后端开发语言。](https://roadmap.sh/backend/languages)

#### 后端开发人员是做什么的？

后端开发人员专注于创建和维护 Web 应用程序的服务器端组件。他们的主要职责是开发服务器端 API、处理数据库操作，并确保后端能够高效地处理高流量。关键职责包括集成外部服务（例如支付网关和云服务），以及提升系统的性能和可扩展性。该职位对于数据处理和安全至关重要，是前端开发人员提供流畅用户体验的坚实后盾。

- 具备扎实的后端开发经验，熟悉微服务和分布式事件驱动架构模式
- 计算机科学或相关学科的学位。
- 具备至少一种后端编程语言（例如 Python、C# 或 Java）的专业经验
- 具备使用 Docker 和容器化技术（例如 Kubernetes）的经验。
- 对持续集成和持续交付有深刻的理解。
- 具备应用领域驱动设计（DDD）、面向对象编程（OOP）等高级设计理念的实践经验。
- 较强的沟通和协作能力。

##### 服务器端开发

这是后端开发人员的核心职责之一。它包括编写服务器端 Web 应用程序逻辑，以处理来自前端的请求、与数据库系统通信以及处理应用程序的业务和后端逻辑。

##### API开发

后端开发人员负责设计、实现和维护应用程序内部不同服务或组件之间以及与外部系统通信所需的 API。这包括创建端点、响应请求以及确保 API 安全。Swagger[是](https://swagger.io/)后端工程师用于设计和编写 API 文档的标准工具。

##### 数据库管理

数据在软件和应用程序开发中扮演着核心角色。后端开发人员决定如何以及在哪里存储这些数据。数据库是最常用的数据存储解决方案之一，后端开发人员负责设计和维护数据库，以实现高效的数据管理。具体任务包括：决定使用哪种数据库（[SQL](https://roadmap.sh/sql)数据库或 NoSQL 数据库，也称为文档数据库）；选择合适的对象关系映射器 (ORM) 将对象映射到数据库；编写高效的查询语句；优化数据库性能；以及确保数据安全。

##### 性能优化

后端工程师不断寻求优化性能的方法。他们会识别性能瓶颈，例如数据库查询缓慢、代码质量差以及应用程序延迟高，并致力于解决这些问题。这包括重构代码以确保代码高质量且可重用、更新依赖项、重新设计数据库模式等等。

##### 第三方服务的集成

有些应用程序会集成第三方网络服务或 API 来执行特定任务。这些第三方服务包括支付网关、云服务（例如 Google Cloud 和[Amazon Web Services）](https://roadmap.sh/aws)以及用户身份验证服务（例如 Auth0）。后端工程师负责将这些外部服务集成到应用程序中。

##### 测试和错误修复

后端工程师编写单元测试、集成测试和端到端测试，以确保系统可靠性。这些测试有助于在持续开发过程中保持系统*稳定运行*且无缺陷。故障排除和修复也是后端工程师的主要职责之一。

##### 文档

后端开发人员负责编写和维护技术规范、API 文档以及面向现有和新团队成员的指南。这些文档是知识传递的途径，可在需要时查阅。

##### 合作

后端工程师与其他团队成员（包括前端开发人员、UI/UX设计师、项目经理、产品经理等）合作，以实现共同目标。

##### 后端开发人员通常参与的项目包括：

- 执行 CRUD 操作的 RESTful API。
- 构建网络爬虫和数据处理服务。
- 建筑图像处理服务。
- 设计和实施内容分发网络（CDN）。
- 数据库系统建模与设计。

##### 掌握至少一种后端编程语言

后端[开发人员](https://roadmap.sh/backend/developer-skills)应该精通至少一种后端编程语言，例如[Java](https://roadmap.sh/java)、C#、[Python](https://roadmap.sh/python)、[Node.js](https://roadmap.sh/nodejs)等。此外，后端工程师如果熟悉一些[后端框架，](https://roadmap.sh/backend/frameworks)例如 Django、[ASP.NET](https://roadmap.sh/aspnet-core)、Ruby on Rails、[Sprint Boot](https://roadmap.sh/spring-boot)等，也会有所裨益。

##### 深入了解数据库和缓存

后端开发人员应该了解数据库和缓存如何在健壮的 Web 应用程序中运作。后端系统会使用多种类型的数据库，包括关系型数据库和文档型数据库。后端系统常用的数据库管理系统包括 Microsoft SQL Server、PostgreSQL、MySQL、MongoDB 等。

[缓存](https://roadmap.sh/guides/http-caching)是将定期请求的数据临时存储起来的过程。它有助于避免重复不必要的数据库查询。Redis 就是一个用于缓存的内存数据存储示例。

##### API知识

API 用于不同系统之间的通信。这些系统可以是微服务、前端和后端系统，也可以是第三方系统。后端开发人员需要了解如何[设计 API](https://roadmap.sh/api-design)并使其可供用户使用。

##### 网络服务器基础知识

后端工程师应该对Web服务器技术有基本的了解。Web服务器负责响应客户端请求。Nginx就是一个典型的Web服务器。在面试过程中，你应该确保候选人理解Web服务器的工作原理，并且知道如何配置它们。

##### 设计模式知识

[设计模式](https://roadmap.sh/guides/design-patterns-for-humans)是可重用的代码库，或用于解决软件组件设计和结构化过程中常见问题的方案。后端开发人员应该了解一些设计模式。常见的设计模式类型包括：

- 创造模式
- 结构模式
- 行为模式
- 建筑模式

##### 熟悉版本控制系统

版本控制系统有助于跟踪和维护应用程序代码的变更历史记录。借助合适的版本控制系统，多个开发人员可以同时协作处理同一个代码库。通过版本控制，他们可以共同维护代码库。

后端工程师需要精通版本控制。Git 是目前最常用的版本控制系统之一。此外，还有许多代码仓库平台，例如 Bitbucket、GitHub 和 GitLab。

##### 了解安全最佳实践

后端开发人员应具备基本的安全实践知识，并实施稳健的安全措施来保护敏感数据，防止未经授权的访问。常见的漏洞包括 SQL 注入、跨站脚本攻击 (XSS) 和跨站请求伪造 (CSRF)。几乎所有大型云服务提供商都提供安全保障功能，例如 AWS IAM、Azure Active Directory 和 Google Cloud Identity。

##### 协作和沟通能力

后端开发人员通常需要与多个团队的其他工程师、经理和测试人员协作，以确保代码一致性并提升用户体验。在面试过程中，您应该密切关注候选人的沟通能力，因为后端开发人员应该能够清晰地阐述问题及其解决方案。您可以向候选人提出一些与协作相关的问题，观察他们的回答。

#### 常见的后端工程师面试问题

**HTTP有哪些不同的方法？**考生应该能够解释不同的HTTP方法，即GET、POST、PUT和DELETE。

**请描述您对持续集成和持续交付的****理解。**应聘者需要用清晰易懂的语言解释 CI/CD 的含义。

**什么是数据库？** **数据库有哪些类型？**应聘者应该了解数据库的定义，以及关系型数据库和文档型数据库的区别，并能举例说明。

**什么是**面向对象**编程？**应聘者应该能够解释面向对象编程（OOP），即围绕对象设计软件的过程。

**请解释同步编程和异步编程的区别。**这里，你应该考察应聘者对同步和异步过程的理解。同步编程按顺序执行任务，每个任务必须完成才能开始下一个任务。异步编程允许任务独立运行，无需等待其他任务完成。它使用 Promise 或 async/await 来处理任务完成情况。

**后端系统中缓存的目的是什么？** **您使用过哪些缓存策略？**应聘者应能解释缓存的含义以及不同的缓存策略，包括：

- 全页缓存
- 数据缓存
- 结果缓存

**Web应用程序中常见的安全漏洞有哪些？** **如何缓解这些漏洞？**应聘者应能解释常见的安全漏洞，例如跨站脚本攻击（XSS）、SQL注入、跨站请求伪造（CSRF）等。可以通过执行适当的输入验证和安全的会话管理来缓解这些漏洞。

#### 有效的面试策略和评估

- **代码审查练习：**提供一段使用任何后端编程语言编写的示例代码片段，并说明其存在的问题或效率低下之处。要求应聘者识别问题、提出改进建议并解释其理由。此练习旨在考察应聘者对细节的关注程度以及对最佳实践的理解。
- **实时编码任务：**使用专注于常见后端开发任务（例如数据库查询或 API 调用）的简短编码挑战。实时编码可以展现候选人在压力下解决问题的能力以及处理实时调试的技巧。
- **系统设计讨论：**提出一个简单的系统设计问题，例如设计一个可扩展的文件存储系统。注意他们解释思路的清晰度、权衡利弊的能力以及对后端基础知识的理解。
- **关于过往项目的行为问题：**询问他们遇到的具体后端挑战以及他们的解决方案。寻找他们适应能力、协作能力和知识深度方面的证据，例如数据库优化或API集成方面的经验。
- **评估沟通能力：**观察他们如何解释技术概念。优秀的候选人能够清晰地沟通，并能根据技术和非技术受众调整解释方式。

#### 了解后端

后端开发人员完全专注于编写应用程序的业务逻辑以及大部分支持逻辑。

也就是说，有些应用场景会将业务逻辑拆分到前端和后端。然而，前端开发人员可能需要在 UI 代码和业务逻辑之间分配时间，而后端开发人员则会将大部分时间集中在核心业务逻辑上。这就是两者之间的主要区别。

#### 精通核心及新兴编程语言

作为后端开发人员，你应该重点关注的最基本技能之一是确定要学习（或至少要留意）的关键编程语言。

业界已将一些基本的后端语言奉为事实标准。这意味着大多数新项目通常都使用其中一种（或多种）编程语言进行编码。

您应该注意的最常见的名字有：

- **JavaScript（或其任何变体，例如 TypeScript）。**这是一个非常常见的选择，因为它也是前端开发人员使用的语言，从而使开发人员更容易在同一个项目的前后两部分进行协作。
- **Python**虽然在其他类型的项目（例如数据处理和[数据科学](https://roadmap.sh/ai-data-scientist)）中非常常见，但在 Web 开发领域仍然非常流行。Python 拥有许多优秀的特性和丰富的支持框架，使其成为编写后端系统的理想选择，而且上手非常容易。
- **Go（又名 Golang）**是一种由 Google 开发的编程语言。它的设计理念是简洁、高效和并发。这使得它在后端开发领域广受欢迎，对于优先考虑性能和并发性的项目来说，它是一个不错的选择。
- **Java**作为企业级解决方案中最常见的选择之一，自 1995 年首次发布以来，一直在不断发展演进。多年来，它凭借其稳健性和不断壮大的开发者社区，赢得了众多大型企业的信赖。虽然 Java 并非最容易学习的语言，但它绝对是十大最受欢迎的[后端语言](https://roadmap.sh/backend/languages)之一（根据[Stack Overflow 2023 年开发者调查](https://survey.stackoverflow.co/2023/#technology-most-popular-technologies)）。

#### 密切关注冉冉升起的新星

如果掌握至少一种最常用的后端语言很重要，那么了解后端领域正在兴起的技术也同样至关重要。

你不会看到每个月都有新的编程语言发布。然而，在短短几年内，你可能会看到好几种语言发布，其中一些可能会长期流行，最终成为新的标准。

有些语言至今仍然非常流行，例如 TypeScript 和 Rust，而另一些语言，例如 Hack 和 Crystal，可能只有少数小众领域的人才知道。

当然，我们无法预测哪种编程语言会成为标准。然而，你需要培养的技能是密切关注行业动态，以便发现新兴趋势。

#### 支持框架的重要性

[不过，特定编程语言的框架](https://roadmap.sh/backend/frameworks)变化速度确实比语言本身的变化速度快得多。

框架旨在为您提供一个简化的入口，让您能够轻松使用完成常见任务所需的功能。例如，在后端 Web 开发中，框架通常负责以下工作：

- **解析 HTTP 请求**并将其转换为您可以轻松交互的对象（这样您就不必学习 HTTP 协议的工作原理）。
- **将请求或响应等概念抽象**成对象和函数，以便从更高层次进行推理。这能让你更容易思考如何使用这些工具解决问题。
- **有了抽象概念，数据访问就变得容易得多。**一些框架提供了所谓的 ORM（对象关系映射）。通过 ORM，您可以与数据库交互，而无需考虑编写 SQL 查询语句甚至数据库模式。
- 还有更多。

框架是后端开发工作中不可或缺的一部分，因此你绝对不能忽视它们。当然，学习和精通所有框架是不可能的。相反，你应该密切关注行业动态，了解哪些框架最常用，然后专注于其中一到两个。

#### 软件设计与架构

编程不仅仅是编写代码。

虽然这种说法可能有点令人困惑，但事实是，软件开发（包括前端和后端开发）中确实涉及大量的架构设计。有时，负责系统架构的这些方面是“架构师”这一特定角色的职责。然而，对于后端系统而言，后端开发人员参与架构讨论和决策也并不罕见。毕竟，你们也在参与设计底层后端基础设施。

从本质上讲，构建系统意味着提出代表解决方案不同方面的概念，然后决定如何使它们相互交互。

为什么架构在这里如此重要？因为它能赋予你代码封装、关注点分离、可重用性等特性，甚至还能带来可扩展性，而这些特性都是架构本身的副产品。

让我们快速了解一下用于创建后端系统的一些最常见架构。

##### **最常见的后端架构**

架构风格和模式太多，无法在一篇文章中全部涵盖，但我们不妨来看看一些最常见的风格和模式，以及它们如何在开发后端系统时为您提供帮助。

- **单体架构：**在单体架构中，整个应用程序被构建为一个单一的、紧密耦合的单元。所有组件（前端、后端、数据库等）都属于同一个代码库。这是一种非常适合初学者的架构，因为它开发起来非常自然，而且如果你的项目规模不会过大，那么你可能不会遇到任何缺点。
- **基于微服务的架构：**应用程序被划分为多个小型、独立的服务，每个服务负责特定的业务功能。这些服务通过 API 进行通信。
- **面向服务的架构：**与微服务类似，面向服务的架构将功能拆分为独立的服务。主要区别在于，这些服务的粒度不如微服务细化，因此它们可能包含与不同业务实体相关的功能。
- **事件驱动架构：**在这种架构中，每个系统（或服务）通过触发操作或进程来响应事件（例如，用户操作和来自其他服务的消息）。所有服务都通过事件总线（也称为“消息总线”）间接相互通信，因此消除了两个或多个服务相互耦合的可能性（这意味着它们不能被单独处理）。
- **无服务器架构：**也称为函数即服务 (FaaS)，无服务器架构让您可以专注于编写代码，而无需担心代码运行所在的服务器。函数会在事件发生时自动执行，无需配置或管理服务器（这些工作都会自动完成）。
- **微内核架构：**这种架构允许您将核心基本功能构建到一个小型微内核中，并将其余功能构建为插件，这些插件可以轻松添加、删除或交换。

如果您想了解更多此处提到的模式和原则，请查看[软件设计和架构路线图](https://roadmap.sh/software-design-architecture)。

#### 精通数据库管理系统

作为后端开发人员，您无疑会在日常工作中处理数据库管理。数据库是存储持久性数据的行业标准。

因此，了解数据库的两大主要类别非常重要：SQL 数据库和 NoSQL 数据库。

##### SQL数据库

这些是标准的结构化数据库（也称为关系型数据库），你需要定义数据模式（本质上就是你正在处理的数据结构），然后使用一种叫做[SQL（结构化查询语言）](https://roadmap.sh/sql)的语言来与其中的数据进行交互。大多数后端开发人员在其职业生涯中都会在某个阶段接触到 SQL 数据库，因为它是最常见的数据库类型。

##### NoSQL数据库

顾名思义，这些并非标准的 SQL 数据库；事实上，此类数据库还包括列式数据库、文档型数据库（例如[MongoDB](https://roadmap.sh/mongodb)）、键值型数据库（例如[Redis](https://roadmap.sh/redis)）等等。它们不使用预定义的数据结构，从而赋予开发者更大的灵活性和控制权，使其能够存储数据的内容和方式。后端开发者通常只会接触到其中的一小部分，因为此类数据库的子类型繁多，而且每年都有新的子类型出现。

这些数据库的一些示例包括：

- MongoDB 是一个基于文档的数据库（如果您感兴趣，请参阅此处的 MongoDB 路线图）。
- Redis，一个内存中的键值对数据库。
- Neo4J，一个图数据库。
- ElasticSearch，一种基于文档的搜索引擎。

最终，SQL 和 NoSQL 之间的选择取决于权衡取舍，以及找出最适合您特定用例的方案。

#### API开发能力

应用程序编程接口（API）无处不在。它们为几乎所有主流系统的后端提供支持（根据[O'Reilly 在 2020 年进行的一项研究](https://www.oreilly.com/pub/pr/3307)，77% 的公司都在使用微服务/API）。

也就是说，如果你想成为一名后端开发人员，你将需要编写 API/微服务代码。因此，理解它们的基础知识对于确保你在该领域的竞争力至关重要。

如果您身处“**系统**”框架内，那么您需要了解如何使用正确的工具与这些 API 进行交互。如果您身处“**外部系统**”框架内，那么您需要了解这些 API 需要遵循的标准类型以及如何实现这些标准。

不过不用担心，在这两种情况下，总有一些框架和库可以用来简化你的任务，并确保你遵循正确的行业标准。

##### 最常见的 API 类型有哪些？

目前业界最常用的 API 类型是 REST 和 GraphQL。

作为后端开发人员，你不需要精通这两种类型，但强烈建议你对其中一种类型有一些实践经验。

- **RESTful API。**这类 API 基于 HTTP 协议，并大量使用 HTTP 动词来赋予每个请求特定的含义。直到最近，它们一直是应用最广泛的 API 类型，因此仍然有很多项目和团队在使用它们。
- **GraphQL** API 也基于 HTTP 协议运行，利用 HTTP 协议及其动词。与传统的 RESTful API 相比，GraphQL 已成为一种强大的替代方案，为数据查询和操作提供了一种灵活高效的方法。GraphQL 允许客户端仅请求所需数据，从而在客户端和服务器之间实现更加定制化和高效的交互。

两者哪个更好？这个问题很难回答，因为它们都能满足您的所有需求。这更多取决于您的具体要求和开发团队的偏好。

#### 精通版本控制

所有后端开发人员（实际上，所有开发人员）都应该掌握的一项必备技能是版本控制，或者换句话说，就是理解和熟悉版本控制系统。

本质上，你需要了解如何使用大家都在使用的版本控制工具。截至撰写本文时，行业标准是[Git](https://git-scm.com/)，虽然可能有些团队还在使用其他（较旧的）工具，但只要你了解当前使用的工具，就完全没问题。

##### 什么是版本控制？

版本控制是指你和其他开发人员在处理同一文件时，能够相互共享代码的功能。

虽然 Git 目前是行业标准，但 GitHub 围绕 Git 创建了一个非常流行的平台，以至于学习 Git 几乎成为一种必然。

所以，赶紧创建一个账号，浏览其他人的作品，并上传你自己的个人项目。这绝对是一个很好的学习方式。

##### 关于 Git，你应该学习哪些内容？

如果你选择 Git 作为你的版本控制系统，那么你应该重点关注以下两个方面。

- **基础知识。**清楚地了解 Git 的工作原理以及添加、推送和拉取更改的基本命令。你应该学习到足够的知识，以便在日常工作中能够熟练地使用它们（因为你肯定会用到）。
- **分支策略。**遗憾的是，仅仅使用 Git 是不够的。虽然通过 Git 可以开始对代码进行版本控制，但当项目足够复杂、团队规模足够大时，单靠 Git 就远远不够了。你需要制定[分支策略](https://learngitbranching.js.org/?locale=es_ES)来组织整个团队的工作流程。

请记住，Git 和 Git 分支并非简单的概念，需要一段时间才能掌握。因此，在学习它们的同时，也务必在使用命令前咨询他人（或使用 ChatGPT 等工具）进行验证。切记，错误的 Git 命令或工作流程可能会给项目带来重大问题，尤其是在多个开发人员共同维护同一代码库的情况下。

#### 测试

对于所有开发人员来说，了解测试是什么以及测试在后端开发工作流程中的重要性至关重要，也是后端开发人员必须掌握的技能之一。

测试是确保代码运行正常的开发过程，它不需要你手动测试每个功能，而是使用工具来测试和重现任何可以通过编程方式发现的问题。

当然，这有助于在测试大型系统时消除潜在的人为错误，并提高这些测试的完成速度（想想看，手动测试只需几秒钟，而手动测试则需要几个小时）。

测试远比我在这里能描述的要复杂得多。你只需要知道，测试系统的方法有很多种，所有后端开发人员都应该了解以下几点：

- **单元测试：**这是最常见的代码测试方法。你需要使用测试框架为代码中每个公开可用的函数/方法编写测试。这样可以确保每一段可用的代码都经过测试，并且能够按预期运行。运行这些测试通常很快，因此你通常会在每次提交代码之前执行它们。
- **集成测试：**如果您正在构建一个由多个独立系统协同工作的系统（例如，基于微服务的架构），那么仅仅测试每个单独的部分是不够的。您还需要确保需要相互交互的系统能够正确交互。这就是集成测试发挥作用的地方。
- **端到端测试 (E2E)：**这类测试类似于集成测试，但它也包含系统的用户界面 (UI)。您可以使用一些工具来自动化 UI 中的操作，模拟真实用户执行这些操作，然后检查结果。例如，点击注销按钮，然后检查是否随后被重定向到登录页面。此流程涉及后端执行一些操作，最终导致用户注销。
- **负载测试：**虽然与之前的测试类型不完全相同，但负载测试对于后端系统来说非常有用，因为它有助于确定后端是否能够处理大量流量。

如果你刚开始接触测试，我建议你暂时只专注于单元测试。一旦你掌握了单元测试，就可以按照上面的图表逐步深入，随着你的进步慢慢学习其他类型的测试。

#### CI/CD 和 DevOps 的熟悉程度

作为后端开发人员，你的代码会不断部署，无论是云环境还是普通的本地服务器。关键在于，你构建的内容都会经过 CI/CD（持续集成和持续部署）流程。

这些流程会自动进行测试（持续集成），并在所有测试通过后自动部署。作为后端开发人员，通常不需要您了解和理解如何配置这些流程；但是，了解这些流程非常重要。

DevOps 是后端开发人员工作的另一个相关领域。当团队规模较小时，后端开发人员可能会被“委派”一些任务，例如配置 CI/CD 流水线、搭建服务器等等。这些任务通常由专门的 DevOps 人员执行。他们的专长是自动化，旨在提高部署效率，并确保所有服务在实际服务器环境中平稳运行。他们在维护应用程序和网站的可靠性和性能方面发挥着至关重要的作用。

因此，虽然这些并非后端开发人员的专属职责，但它们与后端开发人员的日常工作密切相关，所以了解它们很有必要。如果您有兴趣了解更多关于 DevOps 的信息，请查看[我们的 DevOps 路线图，](https://roadmap.sh/devops)其中包含了您成为 DevOps 工程师所需学习的所有关键主题。

#### 软技能

最后，你应该关注的最后一组后端开发技能，实际上并非技术技能，也并非后端开发人员专属。这些技能是每位开发人员在其职业生涯中都应该努力提升的：软技能。

##### 改善沟通

对于任何开发人员来说，与技术人员和非技术人员沟通的能力都是其职业生涯中至关重要的。

对于后端开发人员来说，这一点尤其重要，因为他们要沟通自己的工作及其效果肯定比其他角色（例如前端开发人员）更难，前端开发人员可以实际展示他们正在构建的内容。

作为后端开发人员，你将能够更好地向同事解释问题或障碍，你将能够更有效地进行需求收集，甚至通过更好地向自己阐明问题和潜在解决方案来提高自己的问题解决能力。

##### 批判性思维

作为后端开发人员，磨练批判性思维将有助于你分析复杂问题、更快地识别模式，并为面临的问题提出创新性的解决方案。

突破批判性思维能力的极限，也有助于培养更系统、更具战略性的方法进行编码和构建强大高效的解决方案。

换句话说，它会让你成为更优秀、更高效的程序员。谁不想这样呢？
